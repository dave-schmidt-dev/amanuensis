## Known limitations carried forward into M2+

- Replay-log read APIs (`list_entries`, `get_entry`) have a microsecond consistency window during concurrent writes; docstring at `src/amanuensis/fs/replay_log.py:50-52` currently overclaims "consistent snapshots" — track for docstring downgrade or shared-lock upgrade in M5.x.
- `ReplayLog.get_entry(seq)` is O(num_day_dirs); negligible for Phase 1 substrate sizes but may matter in M5.x cached-call wrapper hot paths.
- Tmp orphans from `atomic_write_text` accumulate (no sweeper); `list_*` paths correctly skip `.tmp.*` but disk usage drifts. Future sweep task.
- `SubstrateSourceMismatch` typed exception not yet created; `add_atom`/`add_relation` raise plain `ValueError` on source-id mismatch. Code reviewer suggested promoting to typed for symmetry; deferred.
- `add_iteration` docstring doesn't yet explain why it takes no `source_id`. Minor polish.
- `docs/schema-reference.md` worked-example block at L508 now has `json` language tag, but the rendered JSON content is intentionally NOT valid parseable JSON (it contains `§` escape that must remain in the canonical bytes; some renderers may complain). Acceptable.
